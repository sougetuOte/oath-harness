# セキュリティレビュー

変更内容に対する包括的なセキュリティ分析を実施する。
`.claude/rules/decision-making.md` の **Critical Agent（批判者）** の視点で評価する。

## 実行ステップ

1. **変更範囲の特定**
   - 変更対象ファイルを列挙
   - 影響を受けるモジュールを特定

2. **セキュリティチェックリスト**
   - [ ] 入力値検証（Input Validation）
   - [ ] コマンドインジェクション対策（bash特有）
   - [ ] シークレット管理（ハードコードされていないか）
   - [ ] ファイルパーミッション（適切な権限設定）
   - [ ] ログ出力（機密情報が含まれていないか）
   - [ ] エラーハンドリング（情報漏洩リスク）
   - [ ] 一時ファイルの安全な取扱い

3. **リスク評価**

| レベル | 基準 | 対応 |
|:-------|:-----|:-----|
| Critical | データ漏洩・システム破壊の可能性 | 即座に修正必須 |
| High | セキュリティホールの可能性 | リリース前に修正 |
| Medium | ベストプラクティス違反 | 計画的に修正 |
| Low | 改善推奨 | 余裕があれば対応 |

4. **報告形式**

```
## セキュリティレビュー結果

### 対象
- [ファイル/モジュール名]

### 発見事項
| # | 重要度 | 項目 | 詳細 | 推奨対応 |
|---|--------|------|------|----------|
| 1 | High   | ... | ... | ... |

### 総合評価
[Pass / Conditional Pass / Fail]

### 次のアクション
- [具体的な対応項目]
```
