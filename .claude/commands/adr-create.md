# ADR作成コマンド

新しい Architecture Decision Record (ADR) を作成する。

## 実行内容

1. 次のADR番号を決定（既存のADRを確認）
2. テンプレートから新しいADRファイルを作成
3. 基本情報の入力を促す

## ADR番号の確認

`docs/adr/` ディレクトリを確認し、次の番号を決定する。

## 基本情報の入力

1. **決定のタイトル**: 簡潔で分かりやすいタイトル
2. **ステータス**: Proposed（提案中）

## 作成手順

1. `docs/adr/YYYY-MM-DD_{title-in-kebab-case}.md` を作成
2. 以下のテンプレートを使用:

```markdown
# ADR-[番号]: [タイトル]

## ステータス
Proposed / Accepted / Deprecated / Superseded

## コンテキスト
この決定が必要になった背景・状況を記述する。

## 決定
何を決定したかを明確に記述する。

## 理由
なぜこの決定をしたのか、検討した選択肢と比較を記述する。

## 結果
この決定による影響（プラス・マイナス両面）を記述する。

## 関連
- 関連するADR、仕様書、Issue等へのリンク
```

## チェックリスト

- [ ] この決定は設計に影響するか？
- [ ] 既存の設計との整合性は確認したか？
- [ ] 関連ドキュメントの更新が必要か？

## 注意事項

- ADRは重要な技術的決定のみに使用（日常的な実装判断は不要）
- 番号は連番で管理（欠番を作らない）
- Deprecatedにする際は必ず後継のADRを指定
